# 매출 분석 (Sales Analysis)

## 한 줄 요약
쇼핑몰의 매출 데이터를 다양한 차원에서 분석하여 매출 성장 기회를 발견하는 스킬

## 상세 설명
매출 분석은 쇼핑몰의 매출 현황과 추이를 다각도로 파악하고, 성장 기회를 도출하는 데이터 분석 역량이다. 이 스킬은 매출 현황 분석(총매출, 기간별, 채널별), 매출 구성 분석(카테고리, 상품, 고객 세그먼트), 매출 Drivers 분석(数量×单价), Trend 및 Seasonality 분석, 예산 대비 실적 분석, 매출 예측 모델링 등을 포함한다. 단순히 숫자를 나열하는 것이 아니라, "왜 그런 결과가 나왔는가"에 대한 통찰을 도출하고, 이를 실행 가능한 전략 과제로 연결하는 능력이 핵심이다.

## 사용 시나리오 예시

**월간 매출 리뷰**
先月の売上報告書を作成し、経営層に報告한다. 총 매출은 전월 대비 12% 증가했으나, 신규 고객 매출은 5% 감소했다. 通过分析發現：新規客の減少は主要広告チャネルのCPA上昇所致と分析された。 이에 따라 低効率チャネルへの投資削減と 신규 채널テスト計画を提出한다.

**매출 드라이버 분석**
이번 분기 매출이 목표에 미달했을 때 원인을 분석한다. 数量と单价に分解して分析した結果、数量は前年同期比+8%だったが、単価は-12%だった。 価格低下の主な原因は、低価格商品の占比增加和促销活动增加所致。これらを踏まえ、下期に高マージン商品の拡充と、プロモーション効率の改善計画を策定した。

## 산출물/템플릿

| 산출물 | 내용 |
|--------|------|
| 매출 현황 리포트 | 기간별, 채널별, 카테고리별 매출 현황 종합 |
| 매출 Drivers 분석 | 수량/단가 기여도, 성장 요인 분해 |
| 매출 추세 분석 | 추세, 시즌성, 이벤트 효과 분석 |
| 매출 예측 보고서 | 향후 3-6개월 매출 예측과 시나리오 |
| 매출 예산 대비 실적 | Budget vs Actual 분석 및 차이 원인 |

**템플릿 구조 예시: 매출 분석 프레임워크**

```
┌─────────────────────────────────────────────────────────────────────────┐
│                      매출 분석 다차원 뷰                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                      매출 분석 차원                              │   │
│  ├─────────────────────────────────────────────────────────────────┤   │
│  │                                                                 │   │
│  │   시간 차원          채널 차원         商品 차원      고객 차원   │   │
│  │   ─────────         ─────────        ─────────    ─────────    │   │
│  │   • 일별            • APP            • 카테고리    • 신규/기존  │   │
│  │   • 주별            • 웹             • 브랜드      • 세그먼트  │   │
│  │   • 월별            • 마켓플레이스    • SKU         • 지역      │   │
│  │   • 분기별          • 오프라인        • 新品/기존    • 채널     │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
├─────────────────────────────────────────────────────────────────────────┤
│  매출 Drivers 분석 (수량 × 단가)                                        │
│                                                                         │
│   매출 증가분 = (수량 증가 기여) + (단가 증가 기여)                      │
│               + (수량×단가 상호작용)                                    │
│                                                                         │
│  예시:                                                                 │
│  • 去年売上: 100億円      今年売上: 115億円    → +15億円 (+15%)          │
│                                                                         │
│  • 数量要因: (今年数量 - 去年数量) × 去年単価                         │
│           = (1,150万個 - 1,000万個) × 1,000円 = +15億円                 │
│                                                                         │
│  • 単価要因: 去年数量 × (今年単価 - 去年単価)                          │
│           = 1,000万個 × (1,000円 - 1,000円) = 0円                      │
│                                                                         │
│  • 結論: 이번 분기 매출 성장은 solely 数量増加所致                      │
│                                                                         │
├─────────────────────────────────────────────────────────────────────────┤
│  매출 추세 분해                                                        │
│                                                                         │
│  원본 데이터                                                         │
│      │    /\      /\    /\                                          │
│      │   /  \    /  \  /  \       추세 + 季節性 + 残差               │
│      │__/    \__/    \/    \____                                     │
│      └─────────────────────────────────────→ 시간                     │
│                                                                         │
│  추세 (Trend): 長期的な増加傾向                                        │
│  季節性 (Seasonality): 月별/주말별 반복 패턴                            │
│  残差 (Residual): 이벤트, エラー 등의 影响                              │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```
