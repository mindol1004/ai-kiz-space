# 매출 분석 (Sales Analysis)

## 한 줄 요약
쇼핑몰의 매출 데이터를 다양한 차원에서 분석하여 매출 성장 기회를 발견하는 스킬

## 상세 설명
매출 분석은 쇼핑몰의 매출 현황과 추이를 다각도로 파악하고, 성장 기회를 도출하는 데이터 분석 역량이다. 이 스킬은 매출 현황 분석(총매출, 기간별, 채널별), 매출 구성 분석(카테고리, 상품, 고객 세그먼트), 매출 드라이버 분석(수량×단가), 추세 및 계절성 분석, 예산 대비 실적 분석, 매출 예측 모델링 등을 포함한다. 단순히 숫자를 나열하는 것이 아니라, "왜 그런 결과가 나왔는가"에 대한 통찰을 도출하고, 이를 실행 가능한 전략 과제로 연결하는 능력이 핵심이다.

## 사용 시나리오 예시

**월간 매출 리뷰**
지난달 매출 보고서를 작성하여 경영진에 보고한다. 총 매출은 전월 대비 12% 증가했으나, 신규 고객 매출은 5% 감소했다. 분석 결과, 신규 고객 감소는 주요 광고 채널의 CPA 상승 때문인 것으로 파악되었다. 이에 따라 저효율 채널 투자를 줄이고 신규 채널 테스트 계획을 제출한다.

**매출 드라이버 분석**
이번 분기 매출이 목표에 미달했을 때 원인을 분석한다. 수량과 단가로 분해하여 분석한 결과, 수량은 전년 동기 대비 +8%였으나 단가는 -12%였다. 가격 하락의 주요 원인은 저가 상품 비중 증가와 프로모션 증가 때문이었다. 이를 바탕으로 하반기에 고마진 상품 확충과 프로모션 효율 개선 계획을 수립한다.

## 산출물/템플릿

| 산출물 | 내용 |
|--------|------|
| 매출 현황 리포트 | 기간별, 채널별, 카테고리별 매출 현황 종합 |
| 매출 드라이버 분석 | 수량/단가 기여도, 성장 요인 분해 |
| 매출 추세 분석 | 추세, 계절성, 이벤트 효과 분석 |
| 매출 예측 보고서 | 향후 3-6개월 매출 예측과 시나리오 |
| 매출 예산 대비 실적 | Budget vs Actual 분석 및 차이 원인 |

**템플릿 구조 예시: 매출 분석 프레임워크**

```
┌─────────────────────────────────────────────────────────────────────────┐
│ 매출 분석 다차원 뷰                                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│ ┌─────────────────────────────────────────────────────────────────┐   │
│ │ 매출 분석 차원                                                   │   │
│ ├─────────────────────────────────────────────────────────────────┤   │
│ │                                                                 │   │
│ │ 시간 차원     채널 차원      상품 차원     고객 차원            │   │
│ │ ─────────     ─────────      ─────────     ─────────            │   │
│ │ • 일별        • APP          • 카테고리    • 신규/기존          │   │
│ │ • 주별        • 웹           • 브랜드      • 세그먼트           │   │
│ │ • 월별        • 마켓플레이스  • SKU        • 지역               │   │
│ │ • 분기별      • 오프라인      • 신상/기존   • 채널               │   │
│ │                                                                 │   │
│ └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
├─────────────────────────────────────────────────────────────────────────┤
│ 매출 드라이버 분석 (수량 × 단가)                                        │
│                                                                         │
│ 매출 증가분 = (수량 증가 기여) + (단가 증가 기여)                       │
│ + (수량×단가 상호작용)                                                  │
│                                                                         │
│ 예시:                                                                   │
│ • 작년 매출: 100억원, 금년 매출: 115억원 → +15억원 (+15%)              │
│                                                                         │
│ • 수량 요인: (금년 수량 - 작년 수량) × 작년 단가                       │
│ = (1,150만개 - 1,000만개) × 1,000원 = +15억원                         │
│                                                                         │
│ • 단가 요인: 작년 수량 × (금년 단가 - 작년 단가)                       │
│ = 1,000만개 × (1,000원 - 1,000원) = 0원                               │
│                                                                         │
│ • 결론: 이번 분기 매출 성장은 전적으로 수량 증가에 기인                │
│                                                                         │
├─────────────────────────────────────────────────────────────────────────┤
│ 매출 추세 분해                                                          │
│                                                                         │
│ 원본 데이터                                                             │
│       /\       /\       /\                                             │
│      /  \     /  \     /  \     추세 + 계절성 + 잔차                   │
│ ____/    \___/    \___/    \____                                       │
│ └─────────────────────────────────────→ 시간                          │
│                                                                         │
│ 추세 (Trend): 장기적인 증가 경향                                        │
│ 계절성 (Seasonality): 월별/주말별 반복 패턴                            │
│ 잔차 (Residual): 이벤트, 오류 등의 영향                                 │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```
