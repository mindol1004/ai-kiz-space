# 카테고리 설계 (Category Design)

## 한 줄 요약
쇼핑몰의 상품 카테고리 구조를 고객 관점에서 체계적으로 설계하고 최적화하는 스킬

## 상세 설명
카테고리 설계는 고객이 원하는 상품을 쉽고 빠르게 찾을 수 있도록 상품 분류 체계를 구축하는 역량이다. 단순한 분류학적 분류를 넘어서 고객의 구매 의사결정 과정을 반영한 논리적 구조를 설계해야 한다. 대형-중형-소형 카테고리 계층, 메타 정보 구조, 필터/정렬 옵션, 크로스 카테고리 연계 등을 포함한다. 고객 세그먼트별 탐색 패턴 분석, A/B 테스트를 통한 UI/UX 최적화, 시즌/트렌드에 따른 카테고리 재편 등 지속적인 개선 능력이 핵심이다.

## 사용 시나리오 예시

**신규 쇼핑몰 카테고리 구조 설계**
뷰티 쇼핑몰 론칭 시, 고객의 구매 여정을 분석하여 초기 카테고리를 설계한다. 스킨케어-메이크업-헤어-바디케어-향수의 대분류를 설정하고, 각 대분류 하에 중분류(예: 기초케어-토너-에센스-크림)를 배치한다. 고객이 상품을 찾는 주요 경로(브랜드별, 기능별, 가격대별)를 분석하여 다중 탐색 경로를 지원한다.

**기존 카테고리 최적화**
고객 이탈률이 높은 카테고리를 분석한 결과, 상품 탐색이 어려워 이탈한 것으로 파악된다. 1depth에서 2depth로 이동하는 이탈률을 측정하고, 불필요한 중복 카테고리를 통합한다. 추천 카테고리와 "함께 많이 구매한 상품" 섹션을 추가하여 크로스 셀링을 강화한다.

## 산출물/템플릿

| 산출물 | 내용 |
|--------|------|
| 카테고리 맵 | 전체 카테고리 구조를 계층적으로 시각화한 맵 |
| 카테고리 정의서 | 각 카테고리명, 포함 상품 범위, SEO 키워드를 포함한 정의서 |
| 카테고리 성과 리포트 | 카테고리별 매출, 마진, 재고 회전율, 전환율 분석 |
| 카테고리 최적화 제안서 | 개선 기회와 실행 과제를 포함한 제안서 |
