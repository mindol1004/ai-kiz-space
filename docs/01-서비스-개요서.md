# 키즈공간 서비스 개요서 (PRD)

## 1. 프로젝트 개요

### 1.1 프로젝트명

**키즈공간** (Kids Space)

### 1.2 프로젝트 목적

부모와 예비 부모를 위한 키즈 전문 쇼핑몰 겸 커뮤니티 플랫폼을 구축하여, 양질의 아동용품 구매와 육아 정보 공유를 하나의 서비스에서 제공한다.

### 1.3 프로젝트 배경

- 키즈 시장은 소비자가 품질과 안전성에 민감하며, 신뢰할 수 있는 정보와 구매 경험이 핵심
- 기존 쇼핑몰은 커뮤니티 기능이 약하고, 커뮤니티 사이트는 구매 연결이 불편함
- 쇼핑과 커뮤니티를 결합하여 "정보 → 구매 → 후기"의 선순환 구조를 만들고자 함

### 1.4 핵심 가치 제안 (Value Proposition)

| 가치 | 설명 |
|------|------|
| **원스톱 경험** | 정보 탐색부터 구매, 후기 공유까지 한 곳에서 해결 |
| **신뢰 기반** | 실 사용자 리뷰와 커뮤니티 검증을 통한 상품 신뢰도 확보 |
| **맞춤 추천** | 자녀 연령/관심사 기반 상품 및 콘텐츠 큐레이션 |
| **소통 공간** | 육아 고민을 나누고 경험을 공유하는 따뜻한 커뮤니티 |

## 2. 타겟 사용자

### 2.1 주요 페르소나

#### 페르소나 1: 김지은 (30세, 워킹맘)

- **자녀**: 3세 아들
- **특징**: 시간이 부족해 효율적인 쇼핑을 원함, 다른 부모의 후기를 참고하여 구매 결정
- **니즈**: 검증된 상품 빠른 탐색, 연령별 추천, 모바일 쇼핑
- **Pain Point**: 여러 사이트를 돌아다니며 리뷰 확인하는 번거로움

#### 페르소나 2: 박서준 (35세, 예비 아빠)

- **상태**: 첫째 출산 예정 (D-3개월)
- **특징**: 육아 용품에 대한 사전 지식이 없어 정보 수집 중
- **니즈**: 출산 준비 체크리스트, 추천 상품 가이드, 초보 부모 커뮤니티
- **Pain Point**: 어떤 제품을 사야 할지 판단 기준이 없음

#### 페르소나 3: 이수아 (28세, 전업주부)

- **자녀**: 5세 딸, 2세 아들
- **특징**: 육아 경험이 풍부하며 커뮤니티 활동에 적극적
- **니즈**: 육아 팁 공유, 중고 거래, 공동구매, 이벤트/기획전 참여
- **Pain Point**: 좋은 정보를 나눌 활발한 커뮤니티 부재

### 2.2 타겟 사용자 요약

| 구분 | 내용 |
|------|------|

| **디바이스** | 모바일 70%, PC 30% (모바일 퍼스트) |
| **지역** | 전국 (온라인 배송) |

## 4. 사용자 스토리

### 4.1 쇼핑 관련

- [x] US-001: 부모로서, 자녀 연령에 맞는 상품을 쉽게 찾고 싶다. 그래야 적합한 제품을 빠르게 구매할 수 있으니까.
- [x] US-002: 구매자로서, 다른 부모들의 실제 사용 후기를 보고 싶다. 그래야 구매 결정에 확신을 가질 수 있으니까.
- [x] US-003: 바쁜 부모로서, 모바일에서 간편하게 주문하고 싶다. 그래야 이동 중에도 쇼핑할 수 있으니까.
- [x] US-004: 구매자로서, 장바구니에 담은 상품을 한 번에 결제하고 싶다. 그래야 여러 상품을 효율적으로 구매할 수 있으니까.
- [x] US-005: 구매자로서, 주문 상태와 배송 현황을 실시간으로 확인하고 싶다. 그래야 언제 도착하는지 알 수 있으니까.

### 4.2 커뮤니티 관련

- [x] US-006: 부모로서, 육아 고민을 다른 부모들과 나누고 싶다. 그래야 도움을 받을 수 있으니까.
- [x] US-007: 경험 많은 부모로서, 육아 팁과 제품 후기를 작성하고 싶다. 그래야 다른 부모들에게 도움이 되니까.
- [x] US-008: 사용자로서, 관심 있는 게시글을 북마크하고 싶다. 그래야 나중에 다시 찾아볼 수 있으니까.
- [x] US-009: 예비 부모로서, 출산 준비에 필요한 정보를 한 곳에서 보고 싶다. 그래야 효율적으로 준비할 수 있으니까.

### 4.3 공통

- [x] US-010: 신규 사용자로서, 소셜 계정으로 간편하게 가입하고 싶다. 그래야 빠르게 서비스를 이용할 수 있으니까.
- [x] US-011: 사용자로서, 새로운 댓글이나 주문 상태 변경 시 알림을 받고 싶다. 그래야 중요한 소식을 놓치지 않으니까.
- [x] US-012: 관리자로서, 상품과 주문을 효율적으로 관리하고 싶다. 그래야 원활한 서비스 운영이 가능하니까.
