## 1. 인증 시스템

### FN-001: 회원가입

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-001 |
| **우선순위** | P0 (Must) |
| **관련 화면** | SCR-AUTH-001 (로그인), SCR-AUTH-002 (회원가입) |

#### 상세 설명

사용자가 이메일 또는 소셜 계정(카카오, 네이버, 구글)을 통해 서비스에 가입할 수 있다.

#### 입력/출력 — 이메일 회원가입

| 항목 | 타입 | 필수 | 유효성 검증 (Zod) | 설명 |
|------|------|------|-------------------|------|
| email | string | O | email(), 중복 확인 API | 로그인 ID |
| password | string | O | min(8).max(20), regex(영문+숫자+특수문자) | bcrypt 해싱 저장 |
| passwordConfirm | string | O | refine(password === passwordConfirm) | 비밀번호 확인 |
| name | string | O | min(2).max(20), regex(한글\|영문) | 실명 |
| nickname | string | O | min(2).max(15), 중복 확인 API | 닉네임 |
| phone | string | O | regex(010-\\d{4}-\\d{4}) | 휴대폰 번호 |
| agreeTerms | boolean | O | literal(true) | 이용약관 동의 |
| agreePrivacy | boolean | O | literal(true) | 개인정보처리방침 동의 |
| agreeMarketing | boolean | X | - | 마케팅 수신 동의 |

#### 소셜 회원가입 절차

1. 소셜 로그인 버튼 클릭 → OAuth Provider 인증
2. NextAuth.js 콜백에서 기존 회원 여부 확인
3. 최초 로그인 시 추가 정보 입력 (닉네임, 휴대폰, 약관 동의)
4. 가입 완료 → 환영 쿠폰 발급 → 홈 또는 callbackUrl로 이동

#### 비즈니스 규칙

| ID | 규칙 |
|----|------|
| BR-001 | 동일 이메일로 중복 가입 불가 |
| BR-002 | 소셜 계정 이메일이 기존 이메일 회원과 동일하면 계정 연동 안내 표시 |
| BR-003 | 14세 미만 가입 시 법정대리인 동의 필요 (Phase 3) |
| BR-004 | 가입 완료 시 환영 쿠폰 (10% 할인) 자동 발급 |
| BR-005 | 비밀번호는 bcrypt(cost=12)로 해싱하여 저장 (security-expert) |
| BR-006 | 모든 사용자 입력은 서버측 Zod 검증 + DOMPurify 산화 (security-expert) |

#### 예외 처리

| ID | 조건 | 메시지 | HTTP |
|----|------|--------|------|
| ERR-001 | 이미 사용 중인 이메일 | "이미 가입된 이메일입니다." | 409 |
| ERR-002 | 이미 사용 중인 닉네임 | "이미 사용 중인 닉네임입니다." | 409 |
| ERR-003 | 비밀번호 조건 미충족 | 구체적 미충족 조건 표시 | 400 |
| ERR-004 | 소셜 인증 실패 | "소셜 로그인에 실패했습니다. 다시 시도해주세요." | 401 |
| ERR-005 | 필수 약관 미동의 | "필수 약관에 동의해주세요." | 400 |

#### 수용 기준

- [ ] AC-001: 이메일 형식이 올바르지 않으면 실시간 에러 메시지가 표시된다
- [ ] AC-002: 비밀번호 입력 시 조건 충족 여부가 실시간으로 체크 표시된다
- [ ] AC-003: 이메일/닉네임 중복 확인 버튼 클릭 시 사용 가능 여부가 표시된다
- [ ] AC-004: 가입 완료 후 자동 로그인되어 홈으로 이동한다
- [ ] AC-005: 환영 쿠폰이 쿠폰함에 자동 등록된다
- [ ] AC-006: 소셜 가입 시 추가 정보 입력 폼이 노출된다

### FN-002: 로그인/로그아웃

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-002 |
| **우선순위** | P0 (Must) |
| **관련 화면** | SCR-AUTH-001 |

#### 상세 설명

이메일+비밀번호 또는 소셜 계정으로 로그인하며, 로그인 상태를 유지하거나 로그아웃할 수 있다.

#### 비즈니스 규칙

| ID | 규칙 |
|----|------|
| BR-007 | 로그인 5회 연속 실패 시 5분간 로그인 차단 (Rate Limiting, security-expert) |
| BR-008 | "로그인 유지" 체크 시 30일간 세션 유지, 미체크 시 브라우저 종료 시 만료 |
| BR-009 | 비밀번호 5회 오류 시 비밀번호 재설정 안내 표시 |
| BR-010 | 로그인 시도는 감사 로그에 기록 (IP, 시간, 성공/실패) (security-expert) |
| BR-011 | CSRF 토큰 검증 필수 (security-expert) |

#### 예외 처리

| ID | 조건 | 메시지 | HTTP |
|----|------|--------|------|
| ERR-006 | 존재하지 않는 이메일 | "이메일 또는 비밀번호를 확인해주세요." | 401 |
| ERR-007 | 비밀번호 불일치 | "이메일 또는 비밀번호를 확인해주세요." | 401 |
| ERR-008 | 로그인 차단 중 | "잠시 후 다시 시도해주세요." | 429 |
| ERR-009 | 정지된 계정 | "이용이 정지된 계정입니다. 고객센터에 문의하세요." | 403 |

#### 수용 기준

- [ ] AC-007: 올바른 자격 증명 입력 시 로그인 성공 후 이전 페이지(callbackUrl)로 이동한다
- [ ] AC-008: 5회 연속 실패 시 로그인 버튼이 비활성화되고 대기 시간이 표시된다
- [ ] AC-009: 로그아웃 시 세션이 삭제되고 홈으로 이동한다
- [ ] AC-010: 소셜 로그인 시 해당 OAuth 제공자 페이지로 리다이렉트된다

## 3. 상품 관리

### FN-004: 상품 카탈로그

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-004 |
| **우선순위** | P0 (Must) |
| **관련 화면** | SCR-SHOP-001 (상품 목록), SCR-SHOP-002 (카테고리별) |

#### 필터 옵션

| 필터 | 타입 | 옵션 |
|------|------|------|
| 카테고리 | 다중 선택 | 대분류/중분류/소분류 (3depth) |
| 가격대 | 범위 슬라이더 | 최소 0 ~ 최대 500,000원 |
| 연령대 | 다중 선택 | 0~6개월, 6~12개월, 1~2세, 3~5세, 6~8세, 9세 이상 |
| 브랜드 | 다중 선택 (검색 가능) | 브랜드 목록 |
| 배송 | 체크박스 | 무료배송만 |
| 할인 | 체크박스 | 할인 상품만 |

#### 정렬 옵션

| 정렬 | 기본값 | 설명 |
|------|--------|------|
| 추천순 | ✓ | 판매량 × 0.4 + 리뷰점수 × 0.3 + 최신성 × 0.3 |
| 신상품순 | | createdAt DESC |
| 판매량순 | | salesCount DESC |
| 가격 낮은순 | | effectivePrice ASC |
| 가격 높은순 | | effectivePrice DESC |
| 리뷰 많은순 | | reviewCount DESC |
| 평점 높은순 | | avgRating DESC |

#### 비즈니스 규칙

| ID | 규칙 |
|----|------|
| BR-017 | 페이지당 20개 상품 표시, 무한 스크롤 (모바일) 또는 페이지네이션 (PC) |
| BR-018 | 품절 상품은 목록 하단 표시, "품절" 배지 + 반투명 처리 |
| BR-019 | 할인 상품은 정가 취소선 + 할인율 배지 표시 |
| BR-020 | 카테고리는 최대 3depth, URL은 최하위 slug 사용 |
| BR-021 | 필터 변경 시 URL 쿼리 파라미터 반영 (브라우저 뒤로가기 지원) |

#### 수용 기준

- [ ] AC-015: 카테고리 필터 선택 시 해당 카테고리 상품만 표시된다
- [ ] AC-016: 가격 범위 슬라이더 조작 시 실시간으로 상품 목록이 갱신된다
- [ ] AC-017: 정렬 변경 시 목록이 즉시 재정렬된다
- [ ] AC-018: 필터 상태가 URL 쿼리에 반영되어 공유/북마크가 가능하다
- [ ] AC-019: 품절 상품은 선택 불가하며 시각적으로 구분된다
- [ ] AC-020: 모바일에서 필터 버튼 터치 시 바텀시트로 필터 UI가 표시된다

### FN-009: 주문 관리

#### 주문 상태 전이 다이어그램

```
[PENDING: 주문접수]
    │
    ▼ (결제 완료)
[PAID: 결제완료] ──────────────────▶ [CANCELLED: 주문취소]
    │                                   (사유 필수, 환불 처리)
    ▼ (관리자 확인)
[PREPARING: 상품준비중] ──────────▶ [CANCELLED: 주문취소]
    │                                   (사유 필수, 환불 처리)
    ▼ (운송장 입력)
[SHIPPED: 배송중] ─────────────────▶ [RETURN_REQUEST: 반품신청]
    │                                   │
    ▼ (배송 완료)                        ▼
[DELIVERED: 배송완료]              [RETURNED: 반품완료]
    │       │                         (환불 처리)
    │       ▼
    │   [RETURN_REQUEST] ──▶ [RETURNED]
    │   [EXCHANGE_REQUEST] ──▶ [EXCHANGED]
    │
    ▼ (수동 확정 또는 7일 자동)
[CONFIRMED: 구매확정]
    (포인트 적립, 리뷰 작성 가능)
```

#### 비즈니스 규칙

| ID | 규칙 |
|----|------|
| BR-046 | 주문 취소: PAID, PREPARING 상태에서만 가능 |
| BR-047 | 반품/교환: DELIVERED 후 7일 이내 신청 가능 |
| BR-048 | 구매 확정: 수동 확정 또는 DELIVERED 후 7일 자동 확정 |
| BR-049 | 구매 확정 시 구매 금액의 1% 포인트 적립 |
| BR-050 | 취소/반품 환불은 원 결제 수단으로 처리 |

#### 수용 기준

- [ ] AC-042: 주문 목록에서 기간별/상태별 필터링이 가능하다

#### 비즈니스 규칙

## 9. 재고 관리

### FN-015: 재고 관리 시스템

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-015 |
| **우선순위** | P0 (Must) |
| **관련 화면** | SCR-ADMIN-011 (재고 관리), SCR-ADMIN-012 (재고 이력) |

#### 상세 설명

관리자가 상품 및 옵션별 재고를 실시간으로 확인하고, 입고/출고/조정/반품 등 재고 변동을 관리할 수 있다. 모든 재고 변동은 이력으로 기록된다.

#### 기능 목록

| 기능 | 설명 |
|------|------|
| 재고 현황 조회 | 상품별/옵션별 현재 재고, 저재고 필터링 |
| 재고 일괄 수정 | 선택 상품 재고 일괄 입력/조정 |
| 입고 등록 | 상품/옵션 선택 후 입고 수량 입력 |
| 출고 등록 | 주문 기반 자동 출고 + 수동 출고 |
| 재고 조정 | 실사 기반 재고 보정 (사유 필수) |
| 재고 이력 조회 | 상품별 입출고 이력, 기간 필터, CSV 내보내기 |
| 저재고 알림 | 설정 임계값 이하 시 관리자 알림 |

#### 비즈니스 규칙

| ID | 규칙 |
|----|------|
| BR-073 | 주문 생성 시 자동 출고(OUT), 취소/반품 시 자동 입고(RETURN) |
| BR-074 | 재고 변동 시 beforeStock, afterStock 기록 필수 |
| BR-075 | 재고 조정(ADJUST) 시 사유 입력 필수 |
| BR-076 | 재고가 0이 되면 해당 옵션 자동 품절 처리 |
| BR-077 | 저재고 알림 임계값: 기본 10개 (상품별 설정 가능) |
| BR-078 | 재고 변동은 트랜잭션으로 처리 (동시성 제어) |
| BR-079 | 모든 재고 변동은 AuditLog + StockHistory 이중 기록 |
| BR-091 | Product.stock = sum(ProductOption.stock) — 옵션별 재고 변동 시 상품 총재고 자동 동기화 |
| BR-092 | 모든 옵션 stock=0 → 해당 옵션 isActive=false 자동 처리 |
| BR-093 | 모든 옵션이 비활성(품절) → Product.status=SOLDOUT 자동 전환 |
| BR-094 | 재고 복원으로 옵션 stock>0 → isActive=true + Product.status=ACTIVE 역전환 |
| BR-095 | 주문 취소 시 StockChangeType=CANCEL로 기록 (반품 RETURN과 구분) |
| BR-096 | 저재고 임계값 도달 시 LOW_STOCK 알림을 ADMIN/SUPER_ADMIN에게 발송 |

#### 예외 처리

| ID | 조건 | 메시지 | HTTP |
|----|------|--------|------|
| ERR-040 | 출고 수량 > 현재 재고 | "재고가 부족합니다." | 422 |
| ERR-041 | 음수 재고 발생 시도 | "재고는 0 미만이 될 수 없습니다." | 422 |
| ERR-042 | 조정 사유 미입력 | "재고 조정 사유를 입력해주세요." | 400 |

#### 수용 기준

- [ ] AC-070: 상품별 현재 재고를 목록에서 확인할 수 있다
- [ ] AC-071: 입고 등록 시 재고가 즉시 증가하고 이력이 기록된다
- [ ] AC-072: 주문 생성 시 재고가 자동 차감되고 출고 이력이 기록된다
- [ ] AC-073: 재고 조정 시 사유 입력이 필수이다
- [ ] AC-074: 저재고 상품만 필터링하여 조회할 수 있다
- [ ] AC-075: 재고 이력을 기간별로 조회하고 CSV로 내보낼 수 있다

## 10. 신고 관리

### FN-016: 신고 시스템

| 항목 | 내용 |
|------|------|
| **기능 ID** | FN-016 |
| **우선순위** | P1 (Should) |
| **관련 화면** | SCR-ADMIN-013 (신고 관리) |

#### 상세 설명

사용자가 부적절한 게시글/댓글/리뷰를 신고할 수 있으며, 관리자가 신고를 검토하고 처리할 수 있다. 동일 대상에 신고 3건 누적 시 자동 블라인드 처리된다.

#### 신고 사유

| 사유 | 설명 |
|------|------|
| SPAM | 스팸/광고 |
| ABUSE | 욕설/비방 |
| INAPPROPRIATE | 부적절한 콘텐츠 |
| MISINFORMATION | 허위/거짓 정보 |
| OTHER | 기타 (상세 사유 필수) |

#### 비즈니스 규칙

| ID | 규칙 |
|----|------|
| BR-080 | 동일 사용자가 동일 대상을 중복 신고 불가 |
| BR-081 | 동일 대상에 신고 3건 누적 시 자동 블라인드 처리 |
| BR-082 | 자동 블라인드: 게시글(isBlinded=true), 댓글(isDeleted=true), 리뷰(status=HIDDEN) |
| BR-083 | 관리자가 신고를 검토(REVIEWED) 또는 기각(DISMISSED) 처리 |
| BR-084 | 본인 작성 콘텐츠는 신고 불가 |
| BR-085 | 신고 처리 시 AuditLog 기록 |

#### 수용 기준

- [ ] AC-076: 게시글/댓글/리뷰에서 신고 버튼을 눌러 사유를 선택하고 신고할 수 있다
- [ ] AC-077: 동일 대상에 중복 신고 시 안내 메시지가 표시된다
- [ ] AC-078: 신고 3건 누적 시 해당 콘텐츠가 자동 블라인드 처리된다
- [ ] AC-079: 관리자가 신고 목록에서 미처리 신고를 확인하고 처리할 수 있다
- [ ] AC-080: 관리자가 기각 처리 시 블라인드가 해제된다
