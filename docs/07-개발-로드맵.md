### ADR-001: 디렉토리 구조 — FSD + App Router 하이브리드

**상태**: Accepted

**컨텍스트**: Next.js App Router의 파일 기반 라우팅과 도메인 기반 코드 분리를 병행해야 한다.

**결정**: FSD(Feature-Sliced Design) 원칙을 App Router 구조에 적용한다.

**테스트 전략**: 각 계층별 테스트 파일은 해당 디렉토리 내 `__tests__/`에 배치한다.

**상태**: Accepted

**컨텍스트**: 코드 품질과 회귀 방지를 위해 테스트 전략이 필요하다.

**결정**: Red-Green-Refactor 사이클을 따르며, 테스트 피라미드를 준수한다.

> **근거**: architecture-expert TDD 원칙, qa-expert 테스트 피라미드

#### 테스트 피라미드

| 계층 | 비율 | 도구 | 대상 |
|------|------|------|------|

## 3. CI/CD 파이프라인

> **근거**: infra-expert GitHub Actions 파이프라인

### GitHub Actions 워크플로우

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 20 }
      - run: npm ci
      - run: npm run lint

  test:
    runs-on: ubuntu-latest
    needs: lint
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm ci
      - run: npm run test -- --coverage
      - run: npm run test:e2e  # Playwright

  build:
    runs-on: ubuntu-latest
    needs: test
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm ci
      - run: npm run build

  deploy:
    runs-on: ubuntu-latest
    needs: build
    if: github.ref == 'refs/heads/main'
    # Vercel auto-deploy via Git integration
```

### 파이프라인 단계

```
[코드 Push] → [Lint] → [단위/통합 테스트] → [E2E 테스트] → [Build] → [Deploy]
    │           │            │                   │              │          │
    │           ▼            ▼                   ▼              ▼          ▼
    │       ESLint       Vitest +           Playwright      Next.js    Vercel
    │       TypeCheck    Coverage 80%+      핵심 플로우      Build     Auto Deploy
```
