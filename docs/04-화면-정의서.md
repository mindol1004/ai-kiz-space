## 2. 공통 컴포넌트 (Atoms)

> **근거**: designer-expert Atomic Design 체계, 컴포넌트 명세 형식

### 2.1 Button 컴포넌트

| 항목 | 내용 |
|------|------|
| **용도** | 사용자 액션 트리거 |
| **카테고리** | Atom |

#### 변형 (Variants)

| 변형 | 설명 | 클래스 |
|------|------|--------|
| primary | 주요 CTA | `bg-primary text-white hover:bg-primary-hover` |
| secondary | 보조 액션 | `bg-white border border-border text-text-primary hover:bg-bg-secondary` |
| ghost | 텍스트 버튼 | `bg-transparent text-primary hover:bg-primary-light` |
| danger | 삭제/위험 액션 | `bg-error text-white hover:bg-red-600` |

#### 크기

| 크기 | 높이 | 패딩 | 폰트 |
|------|------|------|------|
| sm | 32px | 8px 12px | 14px |
| md | 40px | 10px 16px | 16px |
| lg | 48px | 12px 24px | 18px |

#### 상태 (States)

- default → hover → active → focus (ring-2 ring-primary ring-offset-2) → disabled (opacity-50 cursor-not-allowed) → loading (spinner + 텍스트)

#### Props

| Prop | 타입 | 필수 | 기본값 | 설명 |
|------|------|------|--------|------|
| variant | `'primary' \| 'secondary' \| 'ghost' \| 'danger'` | X | `'primary'` | 버튼 스타일 |
| size | `'sm' \| 'md' \| 'lg'` | X | `'md'` | 크기 |
| isLoading | boolean | X | false | 로딩 상태 |
| disabled | boolean | X | false | 비활성 |
| fullWidth | boolean | X | false | 전체 너비 |
| children | ReactNode | O | - | 버튼 텍스트 |
| onClick | function | X | - | 클릭 핸들러 |

#### 접근성

- `role="button"`, `aria-disabled`, `aria-busy` (로딩 시)
- 키보드: Enter/Space로 활성화
- 포커스 링 visible

#### Props

| Prop | 타입 | 필수 | 기본값 | 설명 |
|------|------|------|--------|------|
| label | string | O | - | 라벨 텍스트 |
| type | `'text' \| 'email' \| 'password' \| 'number' \| 'tel'` | X | `'text'` | 입력 타입 |
| placeholder | string | X | - | 플레이스홀더 |
| error | string | X | - | 에러 메시지 |
| helperText | string | X | - | 도움말 텍스트 |

#### 접근성

- `<label>` 연결 (htmlFor), `aria-describedby` (에러/도움말), `aria-invalid` (에러 시)

## 3. 복합 컴포넌트 (Molecules)

### 3.1 ProductCard 컴포넌트

| 항목 | 내용 |
|------|------|
| **용도** | 상품 목록에서 개별 상품 표시 |
| **카테고리** | Molecule |

#### 구조

```
┌──────────────────┐
│  [이미지]          │  ← next/image, aspect-ratio: 1/1, object-cover
│  [할인율 뱃지]     │  ← 좌상단 absolute
│  [위시리스트 ♡]    │  ← 우상단 absolute
├──────────────────┤
│  브랜드명           │  ← text-caption, text-text-tertiary
│  상품명             │  ← text-body-sm, font-medium, line-clamp-2
│  ₩25,000  ₩35,000 │  ← 할인가(font-bold) + 정가(line-through, text-text-tertiary)
│  ★ 4.5 (128)      │  ← text-caption, 별색: secondary
└──────────────────┘
```

#### Props

| Prop | 타입 | 필수 | 설명 |
|------|------|------|------|
| product | ProductSummary | O | 상품 요약 데이터 |
| onWishlistToggle | function | X | 위시리스트 토글 |
| isWished | boolean | X | 위시리스트 여부 |

#### 접근성

- `<article>` 래퍼, 이미지 alt 텍스트 필수
- 위시리스트 버튼: `aria-label="위시리스트 추가"` / `"위시리스트 제거"`
- 전체 카드 클릭 가능 (a 태그 래핑)

### 3.3 SearchBar 컴포넌트

| 항목 | 내용 |
|------|------|
| **용도** | 상품 검색 입력 + 자동완성 드롭다운 |
| **카테고리** | Molecule |
| **컴포넌트 타입** | Client Component ("use client") |

#### 상태

- idle: 검색 아이콘 + 플레이스홀더
- focused: 최근 검색어 + 인기 검색어 드롭다운
- typing: 디바운스 300ms 후 자동완성 드롭다운
- result: 검색어 표시 + X(초기화) 버튼

#### 접근성

- `role="combobox"`, `aria-expanded`, `aria-activedescendant`
- 키보드: ↑↓ 드롭다운 탐색, Enter 선택, Esc 닫기

## 4. 레이아웃 컴포넌트 (Organisms)

### 4.1 Header

| 항목 | 내용 |
|------|------|
| **용도** | 전역 네비게이션 |
| **카테고리** | Organism |
| **컴포넌트 타입** | Server Component (일부 Client: 검색바, 장바구니 뱃지, 프로필 드롭다운) |

#### Desktop (1024px+)

```
┌─────────────────────────────────────────────────────────────────┐
│ [로고]   [쇼핑 ▾]   [커뮤니티]   [기획전]                        │
│                              [🔍 검색바]  [🛒(n)]  [👤 프로필]  │
└─────────────────────────────────────────────────────────────────┘
```

#### Mobile (~767px)

```
┌──────────────────────────────────┐
│ [☰ 메뉴]   [로고]   [🔍]  [🛒(n)]│
└──────────────────────────────────┘
```

- 스크롤 시 `sticky top-0 z-sticky`
- 높이: Desktop 64px, Mobile 56px
- 배경: `bg-bg-primary/80 backdrop-blur-md border-b border-border`

#### 접근성

- `<header>`, `<nav aria-label="메인 네비게이션">`
- 메가 메뉴: `aria-haspopup="true"`, `aria-expanded`
- Skip to content 링크 (sr-only, focus 시 visible)

- `<nav aria-label="하단 네비게이션">`

## 6. 인터랙션 패턴

> **근거**: designer-expert 인터랙션 설계, frontend-expert 에러/로딩/빈 상태 처리

### 6.1 로딩 상태

| 상황 | 패턴 |
|------|------|
| 페이지 초기 로드 | Skeleton UI (콘텐츠 형태 유지) |
| 데이터 추가 로드 | 스피너 (인라인) |
| 버튼 액션 중 | 버튼 내 스피너 + disabled |
| 이미지 로드 | blur placeholder → 선명 (next/image) |
| 무한 스크롤 | 하단 스피너 + Intersection Observer |

### 6.2 빈 상태 (Empty State)

| 상황 | 표시 내용 |
|------|-----------|
| 검색 결과 없음 | 일러스트 + "검색 결과가 없습니다" + 인기 상품 추천 |
| 장바구니 비어있음 | 일러스트 + "장바구니가 비어있습니다" + [쇼핑하기] 버튼 |
| 주문 내역 없음 | 일러스트 + "주문 내역이 없습니다" + [쇼핑하기] 버튼 |
| 게시글 없음 | "아직 게시글이 없습니다" + [첫 글 작성하기] 버튼 |

### 6.3 에러 상태

| 상황 | 패턴 |
|------|------|
| 네트워크 에러 | 토스트 알림 + 재시도 버튼 |
| 폼 유효성 에러 | 인라인 에러 메시지 (input 하단, text-error) |
| 404 | 커스텀 404 페이지 + 홈으로 가기 |
| 500 | 커스텀 에러 페이지 + 재시도/홈으로 가기 |
| 권한 없음 | 403 페이지 또는 로그인 리다이렉트 |

### 6.4 피드백

| 상황 | 패턴 |
|------|------|
| 액션 성공 | 토스트 알림 (우상단, 3초 자동 닫힘) |
| 장바구니 담기 | 토스트 + "장바구니로 이동" 링크 |
| 좋아요/북마크 | 낙관적 업데이트 (즉시 UI 반영 → 서버 동기화) |
| 삭제 확인 | 확인 모달 (title + message + [취소][삭제] 버튼) |
